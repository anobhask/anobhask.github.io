The Axiom Automation Framework is an integrated collection of IT automation software tools assembled together to function cooperatively under a common set of APIs and codebase. Its design follows an explicit "Small, Sharp Tools" philosophy to achieve design simplicity, maximum flexibility and horizontal scalability.

In its current form, all management activity taking place outside a running VM or network device is handled by a cloud suite such as vRealize, Azure, AWS or OpenStack. This includes cloning VM images, provisioning virtual disks and basic network configuration. However, all management activity taking place inside a running VM or network device is handled by Axiom.

Running on Red Hat Enterprise Linux, the Axiom Hubs are the principle server building blocks of the Axiom infrastructure. They are the "distributed workers" in Axiom's overall infrastructure, initiating change and enforcing policy on all managed hosts. The vRA/OpenStack cloud suite talks directly to each hub to deploy VMs in its local datacenter. All managed VMs (also called nodes) are then deployed and managed according to the code residing on that local hub.

To the greatest extent possible, the Axiom Hub has been designed to be a lightweight, self-contained unit that has zero dependencies on other network resources. To increase speed and conserve network bandwidth, each hub only manages nodes that reside in its own local datacenter. An Axiom Master server in each environment (Dev, Stage, Prod, etc.) provides a single point of code deployment for the rest of the Axiom infrastructure. Each hub routinely checks in with its master and updates its local codebase and package repository as appropriate.

 

Functionalities include:

  • Create an automation framework for Sever build and deployment
  • Automate the deployment of servers
  • Automate ORC (Operation Readiness Check) process
  • Automate Server Decommissioning
  • Generate deployment e-mail
  • Create custom workflow in vRO (VMware Orchestrator)
